# Verified lemmas

One page per lemma the Lean kernel accepted. Each entry must include:

- the exact statement;
- the run id and the path to the `.lean` artifact that built;
- the date and which worker produced it.

Nothing here is "proven" on a model's say-so. If the artifact does not build, it
does not belong here. These are the reusable building blocks the formalizer may
draw on in later missions.
